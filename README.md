# zhc-pgsoft-pages

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的集中存放与版本管理用途。

## 项目简介

本仓库主要用于存放各类独立的 HTML 页面文件，适合需要将多个无关联的页面统一托管于同一仓库的场景。所有页面均为静态 HTML，不依赖后端服务，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录说明

```
zhc-pgsoft-pages/
├── pages/           # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/          # 公共资源文件（如样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md        # 本文件
└── index.html       # 可选：仓库首页或导航页
```

- `pages/`：每个 HTML 文件代表一个独立的页面，文件名建议简洁有辨识度。
- `assets/`：存放页面共用的 CSS、JavaScript、图片等资源，可按需组织子目录。
- `index.html`：如希望仓库根目录被访问时显示内容，可在此放置一个导航或说明页面。

## 页面归档说明

- 所有页面均为纯静态 HTML，不包含服务器端逻辑。
- 页面之间无依赖关系，可独立访问。
- 归档的页面可能包含不同的主题、用途或布局，请勿假设它们属于同一项目。
- 若页面使用了外部资源（如 CDN 库），建议在页面内显式引用，避免依赖本仓库内的文件。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。
- 修改页面：直接编辑对应文件，注意保持与公共资源的兼容性。
- 删除页面：移除对应文件及所有相关引用。
- 资源管理：若页面使用了 `assets/` 中的文件，请确保路径一致，避免移动或重命名导致失效。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 中添加或编辑 HTML 文件。
3. 提交并推送至 GitHub。
4. 若启用 GitHub Pages，可配置从 `main` 分支或 `docs/` 目录发布。

## 注意事项

- 本仓库仅用于存储与发布静态页面，不提供任何形式的动态服务或数据接口。
- 请勿将敏感信息（如密钥、密码、个人数据）写入页面或资源文件中。
- 所有页面内容由贡献者自行负责，仓库维护者不对页面内容承担额外责任。
